## How to Use These Examples

1. **Read before your first run** - These examples show exactly what to expect
2. **Reference during execution** - Check expected output against what you see
3. **Troubleshoot issues** - Compare your situation to documented edge cases
4. **Learn variations** - See how to handle common variations and options

---

## Example Structure

Each example follows a consistent format:

1. **Scenario** - What task is being performed
2. **Step-by-step walkthrough** - Commands and expected output
3. **What success looks like** - How to verify completion
4. **Common variations** - Alternative approaches and options
5. **Edge cases** - How to handle unusual situations
